# Print Attendance Report

## Overview
The Print Report feature generates a print-optimized attendance report using the browser's native `window.print()` API. It leverages existing data APIs and CSS variables to create a professional, printable document without external dependencies.

## User Interface

### Access
- **Location**: Settings view → "Report" fieldset
- **Button**: "Print Attendance Report" (primary button with printer icon)
- **Action**: Clicking opens the browser's print dialog with a pre-formatted report

### Visual Design
- Uses existing `--color-primary` for button styling
- Printer icon from Lucide SVG sprite (`#lucide-print`)
- Consistent with other Settings actions (Export CSV, Bulk Import, etc.)

## Technical Implementation

### HTML Structure
```html
<!-- Settings button -->
<button type="button" id="btn-print-report" class="btn-primary">
  <svg class="icon" aria-hidden="true"><use href="#lucide-print"></use></svg>
  Print Attendance Report
</button>

<!-- Hidden print container (only visible during print) -->
<section id="print-report" class="view-hidden" aria-hidden="true"></section>
```

### CSS Print Styles (`@media print`)
Located in `<style>` block (lines ~945-1114):

**Hidden Elements:**
- `.global-header`, `.bottom-nav`, `.controls`, `.attendee-actions`
- `.quick-actions`, `.form-group button`, `.btn-*`, `.modal-overlay`
- `.quick-checkin-form`, `.stats-grid`, `.section-header`, `.recent-activity`

**Page Setup:**
- A4 size, 20mm top/bottom, 15mm left/right margins
- White background, black text, 12pt base font size

**Report Components:**
1. **Header**: "Guild Attendance Register" title + event/generation date
2. **Stats Grid**: 4-column layout (Total Registered, Present, Absent, Attendance %)
3. **Attendance Table**: Student Number | Full Name | Status | Check-in Time
4. **Footer**: Generation attribution + page numbers (Page X of Y)

**Status Badges:**
- Present: Green background (`#d4edda`) with dark green text (`#155724`)
- Absent: Red background (`#f8d7da`) with dark red text (`#721c24`)

### JavaScript

#### `generatePrintReport()` Function
```javascript
function generatePrintReport() {
  const students = dataAPI.getAll();
  const stats = dataAPI.getStats();
  // ... builds HTML string with:
  // - Header with formatted date/time
  // - Stats cards using stats.total, stats.present, stats.absent, stats.percent
  // - Table rows for each student with status badge and check-in time
  // - Footer with generation timestamp
  document.getElementById('print-report').innerHTML = html;
}
```

#### Event Handler
```javascript
document.getElementById('btn-print-report').addEventListener('click', () => {
  generatePrintReport();
  window.print();
});
```

### Data Sources
- **Students**: `dataAPI.getAll()` → Array of `{id, studentNumber, fullName, isPresent, checkInTime, lastModified}`
- **Statistics**: `dataAPI.getStats()` → `{total, present, absent, percent}`
- **Date/Time**: `new Date().toLocaleDateString()` and `.toLocaleTimeString()` (en-GB locale)
- **Utilities**: Reuses existing `escapeHtml()` and `formatTime()` functions

## Browser Compatibility
- Uses standard `window.print()` — supported in all modern browsers
- `@media print` CSS — universal support
- `counter(page)` / `counter(pages)` for page numbers — supported in Chrome, Firefox, Safari, Edge

## Accessibility
- Button has accessible label: "Print Attendance Report"
- Icon marked `aria-hidden="true"`
- Print container has `aria-hidden="true"` (only visible in print context)
- Semantic HTML structure (header, table with thead/tbody, footer)
